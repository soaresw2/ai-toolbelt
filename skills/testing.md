# Testing Skill

This skill enables the AI to write comprehensive tests for code.

## Capabilities

- Write unit tests for functions and classes
- Create integration tests for system components
- Generate end-to-end test scenarios
- Write test fixtures and mock data
- Create test utilities and helpers
- Suggest test cases for edge cases and error conditions
- Review and improve existing tests
- Calculate and improve test coverage

## Testing Frameworks

The AI should be familiar with:
- Jest, Mocha, Vitest (JavaScript/TypeScript)
- PyTest, unittest (Python)
- JUnit, TestNG (Java)
- RSpec, Minitest (Ruby)
- Go testing package (Go)
- And other popular testing frameworks

## Usage

When activated, this skill will:
1. Analyze the code to be tested
2. Identify test scenarios including happy paths and edge cases
3. Write clear, maintainable test code
4. Follow the project's testing conventions
5. Ensure tests are independent and repeatable

## Best Practices

- Write tests that are easy to understand and maintain
- Use descriptive test names
- Follow AAA pattern (Arrange, Act, Assert)
- Mock external dependencies
- Avoid testing implementation details
- Focus on behavior over implementation
